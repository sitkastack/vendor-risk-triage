# Security Policy

This repository hosts reference implementation code that may include patterns related to AI security — prompt injection mitigation, audit logging, secrets handling, and similar. We take security reports seriously, including reports about documentation that could mislead an implementer into building something unsafe.

## Reporting a vulnerability

If you believe you have found a security vulnerability in any code, configuration, or documentation in this repository, please report it privately. Do **not** open a public GitHub issue.

Send a report to **robyn@sitkastack.com** with:

- A description of the issue and where in the repo it appears (file, line, or commit hash)
- Steps to reproduce, if applicable
- Any context on potential impact

## What to expect

- We will acknowledge your report within **3 business days**.
- We will work with you to understand the issue and confirm the scope before any public disclosure.
- Once a fix is in place, we will coordinate timing on disclosure with you and credit you in the release notes unless you prefer to remain anonymous.

## No bug bounty program

We do not currently offer monetary rewards for vulnerability reports. We are grateful for responsible disclosure regardless.
